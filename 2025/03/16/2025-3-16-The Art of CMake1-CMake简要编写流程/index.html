CMake的艺术1-CMake简要编写流程

发表于 2025-03-16 更新于 2025-03-20 分类于 Build Configuration Tools 阅读次数：
本文字数： 2.7k 阅读时长 ≈ 10 分钟

还在使用vs code进行C++的环境的配置吗？想想对那些.vscode文件夹下的一系列文件进行单独配置就感到头大😭，那么是时候来使用CMake来进行自动化编译了，妈妈再也不用担心我的一个.cxx文件一个.vscode文件夹了。当然CMake的作用远不止此，举例来说，为了充分利用每种编程语言的特性，许多大型项目是需要针对不同的功能辅以不同的编程语言实现，这时候就要靠CMake来定制整个编译流程了…

  • CMake靠配置文件CMakeLists.txt来定制整个编译流程。
  • 在CMakeLists.txt所在的目录下运行cmake -B build生成构建系统文件然后再运行cmake --build build来执行构建操作（编译）。
  • 封装Makefile文件来进行定制化make命令。

我们希望在palindrome.cpp中引用#include "MyInclude/aroundCenter.hpp"，同时在main.cpp中引用#include "StrUtils/palindrome.hpp"。这时我们便需要使用add_library()、target_link_libraries()、target_include_directories()和add_subdirectory()这四个命令了。

add_library()

add_library() 是 CMake 中用于定义和创建一个库（静态或动态）的命令。这个命令可以让你在构建系统中创建自己的库，并指定库的源文件。

target_link_libraries()

target_link_libraries()是 CMake 中用于将库链接到另一个目标文件的命令。它用于指定某个目标（如可执行文件或库）所依赖的库，以便在构建时正确链接这些库。

target_include_directories()

target_include_directories()是CMake中用于为特定目标（可以是可执行文件或库）指定头文件搜索路径的命令。这些路径会告知编译器在编译目标时到哪里去寻找头文件。

add_subdirectory()

add_subdirectory()是CMake中用于在构建树中包含一个子目录。这个命令将子目录中的 CMakeLists.txt 文件包含到主目录的构建过程中，并且子目录会作为独立的构建单元进行处理。

下面让我详细解释一下可执行文件Demo的创建所需要的函数：

  1. add_executable() 是创建可执行文件的必需命令
  2. target_link_libraries() 是可选的，只在需要使用其他库时才需要
  3. 如果使用 target_link_libraries()，必须在 add_executable() 之后
  4. 它们是独立的命令，但在需要使用外部库时会配合使用

构建可执行文件的两阶段：

  1. 编译阶段（Compile）：将源代码（如 .c 或 .cpp 文件）转换为目标代码（即 .o 文件或 .obj 文件）。在此阶段，编译器仅检查代码的语法、语义以及库中函数的声明（即头文件中的函数声明），但不会关心函数的实际定义是否存在。

  2. 链接阶段（Link）：将各个编译好的目标文件和库文件链接在一起，形成最终的可执行文件。在这个阶段，所有函数的实际实现（定义）必须找到，链接器会将目标代码与库文件中的实现结合起来，确保可执行文件可以正常运行。
